📚 Timetable Extraction System

AI-powered teacher timetable extraction system using GPT-4 Vision for intelligent parsing of schedule documents.

🎯 Current Status: Production-ready prototype with complete backend API and modern web UI.
Try Demo View GitHub

🚀 Installation

Prerequisites

  • Node.js 18 or higher
  • npm or yarn
  • OpenAI API key

Setup Steps

# Clone repository
git clone <repository-url>
cd timetable-extractor

# Install dependencies
cd backend
npm install

# Configure environment
cp .env.example .env
# Edit .env and add your OPENAI_API_KEY

# Start server
npm start
✅ Success! Server running at http://localhost:4012

⚡ Quick Start

1. Upload via Web Interface

Open http://localhost:4012 and drag & drop a timetable file.

2. Upload via API
curl -X POST http://localhost:4012/api/extract \
  -F "file=@timetable.png"
3. View Results

Get structured JSON with time blocks, metadata, and confidence scores.

🔌 API Endpoints

POST /api/extract

Upload and extract timetable data from an image or PDF.

Parameter Type Description
file File Timetable image (PNG, JPEG) or PDF

Response (200 OK)

⚠️ Error Handling

Error Code HTTP Status Description
UNSUPPORTED_FILE_TYPE 415 File type not supported
FILE_TOO_LARGE 413 File exceeds 10MB limit
VALIDATION_FAILED 422 Extracted data invalid
PROCESSING_FAILED 500 Internal processing error

🛠️ Technology Stack

Backend
  • Runtime: Node.js 18+
  • Framework: Express.js
  • LLM: OpenAI GPT-4o
  • PDF Processing: pdf-parse
  • OCR: Tesseract.js
  • Validation: Zod
  • Logging: Winston
Frontend
  • Current: Pure HTML/CSS/JavaScript
  • Recommended: React + TypeScript
  • Styling: TailwindCSS
  • Calendar: FullCalendar

🤖 LLM Integration

GPT-4 Vision Strategy

The system uses GPT-4o with vision capabilities for intelligent extraction:

  • Model: gpt-4o
  • Temperature: 0 (deterministic)
  • JSON Mode: Enabled for structured output
  • Max Tokens: 4096

Prompt Engineering

Multi-shot prompting with structured output:

  • System prompt defines expert role
  • Few-shot examples for context
  • Handles vertical text (B R E A K → Break)
  • Preserves original subject names
  • Includes confidence scoring
💡 Tip: The LLM handles complex layouts, handwritten text, and spaced characters automatically.

💾 Database Schema

While the prototype doesn't use a database, here's the recommended schema for production:

timetables
  - id (UUID)
  - teacher_name
  - class_name
  - term
  - school_name
  - uploaded_at
  - extraction_confidence

timeblocks
  - id (UUID)
  - timetable_id (FK)
  - day_of_week
  - start_time
  - end_time
  - duration_minutes
  - subject
  - subject_type
  - notes
  - confidence
